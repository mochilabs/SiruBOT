## 2025-02-23 - Screen Reader Context for "Read More" Links
**Learning:** Found a common accessibility anti-pattern in the dashboard's card components where multiple links shared the exact same text ("보러가기 →" / "Go see →"). While visually distinct due to card placement, screen readers listing links would just read "Go see, Go see, Go see". Additionally, decorative text characters like "→" are often read aloud confusingly ("right pointing arrow") by screen readers.
**Action:** Applied a reusable pattern for card links: Use `aria-label` to provide the full context (e.g., "인기 곡 순위 보러가기" / "Go see popular tracks") and wrap decorative arrows in `<span aria-hidden="true">` to prevent screen reader noise. This pattern should be applied to all generic-text links across the dashboard.
